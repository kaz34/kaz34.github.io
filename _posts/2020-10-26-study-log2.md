### 課題

cloneしたときに、

また

```jsx
$ docker-compose run web rake db:create
```

と

```jsx
$ docker-compose up
```

をしなければ、いけなかった。

### 解決すること

表示するだけならできるが、DBが絡んで来るから、

DB、セーブデータを保存する必要がある。

↓

volumeを作成する

```jsx
docker volume create ボリューム名
```

```jsx
docker run -dp 3000:3000 -v ボリューム名:/etc/hoge イメージ名
```

↓

何かファイル内で変更、保存後、コンテナを削除してみる。

↓

コンテナを再びrunする。

↓

変更がそのまま残っている。

↑

これらは、qiitaのdocke-composeが全て実行してくれていた。
